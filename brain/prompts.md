# ⚙️ PROMPTS — Paolo AI Brain

## CORE LOGIC

Ogni contenuto deve seguire:

VISIVO → HOOK → PROBLEMA → DIREZIONE → STOP → CTA IMPLICITA

Mai:
- spiegare tutto
- chiudere completamente il discorso

---

# 🧩 MODULAR SYSTEM

## 1. HOOK BLOCK

Genera un hook diretto, tagliente, eventualmente ironico.

Linee guida:
- massimo 1–2 frasi
- deve creare attrito o curiosità
- può contraddire una pratica comune

Esempi struttura:
- "Usare un font non è progettare un logo."
- "Sembra funzionare. Non funziona."
- "Questo è il problema che nessuno considera."

---

## 2. PROBLEM BLOCK

Descrivi il problema reale del cliente o del settore.

Regole:
- concreto, non teorico
- legato a situazioni reali
- deve far riconoscere il problema a chi legge

---

## 3. INSIGHT BLOCK

Spiega la direzione, NON la soluzione completa.

Regole:
- far capire che esiste una soluzione progettuale
- NON spiegare come replicarla
- introdurre il valore del disegno/manualità

---

## 4. VISUAL CONTEXT BLOCK

Descrivi cosa mostra il visual (progetto finito).

Regole:
- collegare il visual al problema
- evidenziare le scelte senza spiegarle tutte

---

## 5. CTA BLOCK

CTA implicita o minima.

Esempi:
- "Se ti trovi in questa situazione, sai cosa fare."
- "Se il tuo brand ha questo problema, parliamone."
- "Questo è il tipo di lavoro che faccio."

---

# 📱 INSTAGRAM — CAPTION CORE

Scrivi una caption Instagram seguendo questo schema:
- hook diretto

- insight (senza spiegare tutto)
- collegamento al progetto visivo
- chiusura aperta con CTA implicita

- spiegazioni passo passo

# ⚡ POST POLARIZZANTI

Prompt:

Scrivi un contenuto breve e diretto che metta in discussione una pratica comune nel design (es. uso dei font nei loghi).

Struttura:
- affermazione forte iniziale
- breve spiegazione
- apertura finale

Tono:
critico ma non aggressivo, con ironia controllata

far percepire competenza, non insegnare
